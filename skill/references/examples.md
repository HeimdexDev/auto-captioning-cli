# 좋은 예 / 나쁜 예 — 시공간 흐름 캡션

키프레임은 **시간 순서**(첫 장 = 이른 시점, 끝 장 = 늦은 시점)다. 캡션은 프레임 "사이"의 변화를 설명한다.

---

## 케이스 1: 인물이 이동하며 동작 (프레임 2~3장)

**상황**: 프레임1 — 여성이 화면 왼쪽에 서서 정면을 본다. 프레임2 — 중앙으로 이동해 한 손을 들어 올린다. 자막 없음.

| 평가 | ref | 캡션 | 사유 |
| --- | --- | --- | --- |
| ✅ | ref_a (행동) | "여성이 처음에 정면을 보고 서 있다가 이어서 중앙으로 옮겨 한 손을 들어 올린다" | 동작 순서 명확 |
| ✅ | ref_b (공간·시점) | "왼쪽에 있던 인물이 다음 프레임에서 화면 중앙으로 이동하고 구도는 그대로 유지된다" | 위치 변화 강조 |
| ✅ | ref_c (요약) | "여성이 왼쪽에 서 있는 장면으로 시작해 중앙에서 손을 드는 동작으로 마무리된다" | 시작→끝 압축 |
| ❌ 나쁨 | | "여성이 무대를 가로질러 걸어가 춤을 춘다" | 프레임 사이 경로·춤 날조 |
| ❌ 나쁨 | | "활기차게 이동하는 예쁜 여성" | 평가어 |

```json
"shot_type": "full", "motion_type": "subject_motion", "temporal_change": "major",
"text_on_screen": null, "mood": ["casual"]
```

## 케이스 2: 카메라가 줌인 (인물은 거의 정지)

**상황**: 프레임1 — 인물 전신 wide. 프레임2 — 같은 인물 상반신으로 가까워짐(줌인).

| 평가 | ref | 캡션 |
| --- | --- | --- |
| ✅ | ref_a (행동) | "인물은 거의 같은 자세를 유지하고 두 프레임에 걸쳐 큰 동작 변화는 없다" |
| ✅ | ref_b (공간·시점) | "전신을 담던 화면이 다음 프레임에서 상반신 위주로 줌인되어 인물이 크게 보인다" |
| ✅ | ref_c (요약) | "같은 인물을 와이드에서 점차 가까이 당겨 보여주는 줌인 위주의 장면" |

```json
"shot_type": "medium", "motion_type": "camera_motion", "temporal_change": "minor",
"text_on_screen": null, "mood": null
```

`shot_type`은 대표 거리(여기선 끝 프레임의 medium), 거리 변화 자체는 `motion_type: camera_motion`.

## 케이스 3: 화면 속 사물이 움직임 (object_motion)

**상황**: 프레임1 — 책상 위에 상자가 닫혀 있다. 프레임2 — 손이 들어와 뚜껑이 열려 있다. 사람 얼굴은 화면 밖.

| 평가 | ref | 캡션 |
| --- | --- | --- |
| ✅ | ref_a (행동) | "화면 밖에서 들어온 손이 닫혀 있던 상자의 뚜껑을 열어 안이 드러나게 한다" |
| ✅ | ref_b (공간·시점) | "책상 중앙의 상자는 자리를 지키고 두 번째 프레임에서 뚜껑만 위로 열린다" |
| ✅ | ref_c (요약) | "닫힌 상자가 손에 의해 열리는 과정을 가까이서 담은 장면" |

```json
"shot_type": "closeup", "motion_type": "object_motion", "temporal_change": "major",
"text_on_screen": null, "mood": null
```

## 케이스 4: 거의 변화 없는 정적 씬 / 단일 프레임

**상황**: 프레임 1~2장이 사실상 동일 — 인물이 정면을 보고 말하는 토크 샷.

| 평가 | ref | 캡션 |
| --- | --- | --- |
| ✅ | ref_a (행동) | "인물이 정면을 향해 앉아 말하는 자세를 두 프레임 내내 거의 그대로 유지한다" |
| ✅ | ref_b (공간·시점) | "상반신이 화면 중앙에 고정되어 구도와 거리가 프레임 간 변하지 않는다" |
| ✅ | ref_c (요약) | "구도 변화 없이 인물이 정면을 보고 이야기하는 정적인 토크 장면" |

```json
"shot_type": "medium", "motion_type": "static", "temporal_change": "none",
"text_on_screen": null, "mood": ["calm"]
```

단일 프레임이면 `notes`에 "키프레임 1장, 시간 정보 없음".

| 평가 | ref | 캡션 |

## 케이스 6: 자막이 프레임 간 바뀜

**상황**: 인물 구도는 그대로인데 하단 자막이 프레임1 "29,000원" → 프레임2 "한정수량"으로 바뀐다.

| 평가 | ref | 캡션 |
| --- | --- | --- |
| ✅ | ref_b (공간·시점) | "인물 구도는 유지된 채 하단 자막이 가격에서 다른 문구로 바뀐다" |

```json
"shot_type": "medium", "motion_type": "static", "temporal_change": "minor",
"text_on_screen": ["29,000원", "한정수량"], "mood": null
```

자막 내용은 `text_on_screen`에. 캡션에서는 "자막이 바뀐다"처럼 시각 사실로.

## 자주 마주치는 함정

### 함정 1: 프레임이 1장뿐
→ 시간 정보 없음. 공간 구성만 묘사, `motion_type: static`, `temporal_change: none`, `notes`에 기록.

### 함정 2: 변화를 날조
→ 2~3장 사이의 보이지 않는 중간 동작 추가 금지. 보이는 변화만.

### 함정 3: 사람·카메라가 둘 다 움직임
→ `motion_type`은 더 두드러진 쪽 하나. 캡션에는 둘 다 적어도 됨.

### 함정 4: 제품 정보에 매몰
→ 이 데이터셋은 흐름 이해용. 제품명·가격은 핵심이 아니다(보이면 `text_on_screen`).
